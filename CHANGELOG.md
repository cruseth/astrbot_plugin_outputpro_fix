# CHANGELOG

## v2.2.3-fix.5

错误修复：

- 修复 `\s` 空白分段符在中文空格、全角空格、tab、回车组合等场景下优先级不稳定的问题。
- 保留英文单词间半角空格保护，避免配置 `\s` 后把英文短语按单词拆开。
- 分段最大数量配置开放 `0`，表示不限制分段数量，适合需要所有空格/回车命中都稳定分段的场景。

## v2.2.3-fix.4

错误修复：

- 增强主动发送拦截兼容性：当平台或其他插件更新/替换 `context.send_message` 后，OutputPro 会重新安装包装器。
- 新增对事件级 `event.send()` 主动发送的处理，修复部分插件消息绕过 OutputPro pipeline 的问题，同时避免重复处理 AstrBot 核心被动回复。
- 智能引用改为按 `unified_msg_origin` 隔离消息队列，并兼容不同来源的消息组件类型，减少平台更新后引用失效或串会话的问题。

## v2.2.3-fix.3

错误修复：

- 修复分段发送后 AstrBot 日志和上下文只保留最后一段的问题。
- 分段步骤不再在装饰阶段主动发送前置分段，而是将完整分段链回填给 AstrBot 记录。
- 新增一次性分段发送包装器，在实际发送阶段按分段计划逐段发送，同时避免与 AstrBot 内置逐组件发送重复拆分。

## v2.2.3-fix.2

错误修复：

- 主动发送消息现在也会经过 OutputPro pipeline，修复定时推送、后台任务等直接调用 `context.send_message()` 时绕过合并转发等处理的问题。
- 增加主动发送递归保护，避免 `split` 等步骤内部再次发送消息时重复进入 pipeline。

## v2.2.3

新增特性：

- 引入可配置的拆分优先级，在选择文本拆分点时优先考虑特定字符。
- 为纯文本片段新增前置空白字符裁剪逻辑，同时在提及组件（At 组件）之后保留单个空格。

错误修复：

- 当消息链为空或不以纯文本组件结尾时，避免执行转发处理。

优化与改进：

- 改进拆分点选择逻辑，在遵循每个 token 的拆分优先级的同时平衡各段长度。
- 简化片段构建逻辑，并确保回复/提及前缀始终一致地附加到片段上。
- 调整打字延迟计算，使用明确的逐字符延迟、最小延迟和最大延迟参数。
- 修改平台相关的转发逻辑，改为直接基于平台名称分支处理，覆盖 aiocqhttp 和 Telegram。
- 统一插件数据目录解析方式，使其与 AstrBot 核心数据路径工具保持一致，并在配置中暴露插件名称。

## v2.2.2

Enhancements:

- 在分词过程中发出拆分 token 时，忽略仅由空白字符组成的分隔符匹配结果。
- 使用可配置的最小值和最大值来限制单字符延迟，而不是使用固定常量。
- 允许配置结尾标点字符，并在未提供任何结尾标点时禁用结尾裁剪。
- 在构建拆分模式时，对字面量换行符和转义换行符进行统一处理。
- 在选择拆分点时，去除最小片段长度的限制。

## v2.2.1

使用全新的分词器/构建器流水线重写消息拆分步骤，并简化拆分行为的配置。

Enhancements:

- 引入一个分词器，在保护颜文字的前提下，执行对括号和引号安全的文本分词，并支持可配置的拆分分隔符。
- 将分段构造重构为一个专用的构建器，用于管理段落长度限制、前缀组件以及结尾标点的清理。
- 通过用基于字符列表的拆分正则表达式和打字速度参数，替代多个计时和分隔选项，从而简化拆分配置。
- 将打字延迟计算调整为基于可配置“每秒字符数”的确定性逐字符模型。

## v2.2.0

New Features:

- 新增错字模拟步骤，支持单字同音替换、整词同音替换、错误声调候选与按概率追加“正确字/词”提示。
- 在分段步骤中新增智能分段模式，支持更自然的短句切分与动态延迟。

Enhancements:

- 新增分段字数上限，消息超过设定长度时可直接跳过分段。
- 让最大分段数限制同时作用于普通分段和智能分段。
- 优化纠正提示与智能分段的协作方式，确保换行分隔的纠正内容不会被重新合并。
- 当消息超过分段字数上限时，错字模拟不再追加纠正提示，避免输出结构失真。
- 在转发的错误报告中增加群组 ID、用户 ID、用户输入、匹配到的关键词以及原始错误文本，以提升管理员对问题的可追踪性和上下文理解。

## v2.1.7

为 Telegram 平台新增长消息和自动撤回支持，扩展 TTS 以支持可插拔模型提供方和跨平台 QQ 语音中继，并优化消息拆分、空格保护以及智能回复行为。

New Features:

- 在转发步骤中，通过可展开的引用块支持 Telegram 长消息处理。
- 支持可配置的 TTS 模型提供方，并在需要时回退到 QQ 语音聊天中继，包括跨平台语音中继能力。

Bug Fixes:

- 确保在拆分步骤中丢弃前导的空内容，同时避免产生空的最终结果。
- 修复智能回复队列处理，通过对字符串化的消息 ID 进行操作并避免清空整个队列。

Enhancements:

- 改进 TTS 行为，将音频记录构造集中处理，并支持多种音频 URL/文件格式。
- 在消息拆分时规范化和清理前导空片段，同时保留提及（@）和回复信息。
- 调整空格保护逻辑，将仅包含 CJK 的空格视为普通空格，并保护其他空格模式。
- 增强智能回复触发逻辑，以支持更多消息组件类型，并跟踪机器人回复标记，以便更好地插入引用。
